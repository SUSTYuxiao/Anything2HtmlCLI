# Atlas X1 — AI 时代的产品发布

> 2026 春季发布会｜AI 工作站事业部

---

## 1. 我们为什么再造一台 AI 工作站

过去十年, "PC" 与 "AI" 始终是两条平行轨道。本地算力没跟上模型膨胀, 云端 API 又把延迟与隐私甩给用户。

- 78% 的开发者每天使用 AI, 但只有 12% 在本地推理
- 平均每次云端 round-trip 延迟 1.4s, 远高于人类感知阈值 200ms
- 我们要把这条断层焊起来

## 2. Atlas X1 一句话

**一台为 Agent 工作流而生的桌面 AI 工作站**——本地 70B 模型实时推理, 全栈隐私, 价格 $4,999。

## 3. 三个核心数据

- 70B 参数模型本地 35 tokens/s 持续输出
- 上下文窗口 1M tokens, 一次吞下整个 monorepo
- 24 小时连续推理 待机噪音 28dB, 安静过图书馆

## 4. 硬件: NPU 不是 GPU 的副产品

我们没有用现成的桌面 GPU。Atlas X1 自研了 **N1 NPU**——为 transformer 推理特化的 ASIC。

- 192GB unified memory, 800GB/s 带宽
- 集成 Flash Attention 2 + Speculative Decoding 硬件加速
- 单芯片功耗 180W, 是同等吞吐桌面 GPU 的 1/3

## 5. 软件: AgentOS 1.0

硬件只是底座。Atlas X1 真正的差异化在 **AgentOS 1.0**——一个把 Agent 当一等公民的操作系统层。

- 系统级 sandbox: 每个 Agent 默认零网络 / 零文件 / 零 shell, 显式申请
- 原生 MCP 支持: 系统 daemon 即 MCP server, 任意 Agent 即插即用
- 全局上下文图谱: 跨应用的语义记忆, 不再 copy-paste

## 6. 性能基准: 数字会说话

我们在三个真实工作流上对比了 Atlas X1 与 RTX 5090 + 云 API 混合方案。

- 大型代码库重构 (2.3M LOC): 本地 18 分钟 vs 云端 47 分钟
- 100 篇论文综述 (Deep Research): 本地 6 分钟, 零网络费用
- 24h 长任务 (autonomous coding): 完成度 94% vs 云端 73% (受限于上下文截断)

## 7. 隐私: 默认本地, 显式上云

我们见过太多 "AI 优先" 的产品把用户数据当训练料。Atlas X1 反过来。

- 所有模型权重 + 上下文默认留在 NPU 内存
- 上云需要用户每次显式确认, 系统级红色横幅提示
- 出厂自带审计日志: 哪个 Agent 在何时读了哪个文件, 完整可追溯
